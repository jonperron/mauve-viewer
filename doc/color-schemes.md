# Color Schemes

Mauve Viewer offers several color schemes for Locally Collinear Blocks (LCBs). Each scheme highlights a different property of the alignment. You can switch schemes using the color scheme selector in the toolbar.

## Standard Schemes

### LCB Color

The default scheme. Each LCB receives a unique color generated by rotating the hue in the HSB color space. This makes it easy to track individual blocks across genomes and spot rearrangements.

### Offset

Colors each LCB based on its generalized offset — the displacement of the block relative to a reference position. Blocks that appear in similar genomic contexts across genomes receive similar colors. This helps identify likely collinear regions.

### Normalized Offset

Same concept as Offset, but the offset values are rank-ordered and evenly distributed across the color spectrum. This produces better color separation when offset values are unevenly distributed.

### Multiplicity

Colors each LCB based on the number of genomes that share it. Blocks present in all genomes have one color, blocks present in a subset have another, and so on. This highlights core (shared) versus accessory (subset-specific) genome regions.

### Multiplicity Type

Colors each LCB based on the exact combination of genomes that share it. Each unique genome combination receives a distinct color. This scheme is available only when the alignment contains 62 or fewer sequences, because the number of possible combinations grows exponentially.

### Normalized Multiplicity Type

Same concept as Multiplicity Type, but unused genome combinations are removed and the remaining patterns are evenly distributed across the color spectrum. This avoids wasted color space for combinations that do not appear in the alignment.

## Backbone Schemes

These schemes are available only when backbone data is loaded alongside the alignment.

### Backbone LCB

Regions conserved across all genomes (backbone) are colored mauve (#9370DB). Regions conserved in a subset of genomes are colored using the Multiplicity Type scheme. This provides a clear visual distinction between core and accessory regions.

### Backbone Multiplicity

Segments conserved across all genomes are always colored mauve (#9370DB). Other conservation patterns receive maximally distinct colors based on the genome presence/absence bitmask. This helps distinguish different conservation patterns when many multiplicity types are present.

## Dynamic Menu

The color scheme menu adapts to the loaded data:

- **Backbone schemes** are hidden when no backbone data is loaded or when the alignment contains more than 62 sequences.
- **Multiplicity Type** and **Normalized Multiplicity Type** are hidden when the alignment contains more than 62 sequences.
