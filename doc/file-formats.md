# File Formats

Mauve Viewer supports several input formats for sequence alignments, annotations, and auxiliary data. Files are detected automatically by their extension.

## Alignment Formats

| Format | Extensions | Description |
|--------|-----------|-------------|
| XMFA | `.xmfa`, `.alignment` | Extended Multi-FASTA Alignment. The primary alignment format containing aligned sequence blocks with strand orientation, genome metadata, and optional annotation file references. |
| Mauve | `.mauve`, `.mln` | Compact alignment format produced by the Mauve aligner. Contains anchor coordinates without full sequence data. Converted internally to XMFA for display. |
| JSON LCB | `.json` | JSON representation of Locally Collinear Block (LCB) coordinates with left/right endpoints per genome. |

## Sequence Formats

The following sequence formats are recognized by the parser but cannot be loaded as standalone input files. They are used internally when referenced by XMFA alignment headers.

| Format | Extensions | Description |
|--------|-----------|-------------|
| FASTA | `.fasta`, `.fa`, `.fna`, `.faa` | Standard multi-entry sequence format. Multiple entries are concatenated with contig boundaries recorded. |
| Raw | `.raw` | Plain nucleotide text without headers. |

## Annotation Formats

| Format | Extensions | Description |
|--------|-----------|-------------|
| GenBank | `.gbk`, `.gb`, `.genbank` | NCBI GenBank flat file format. Provides gene annotations including CDS, tRNA, rRNA, and misc_RNA features with qualifiers such as locus_tag, gene, product, and protein_id. |
| EMBL | `.embl` | EMBL flat file format. Similar annotation content to GenBank in a different layout. |
| INSDseq XML | `.xml`, `.insdc` | NCBI INSDseq XML format for sequence and annotation records. |

## Other Formats

| Format | Extensions | Description |
|--------|-----------|-------------|
| Newick | (parsed from embedded data) | Phylogenetic tree format used for guide trees in progressive alignment. |

## Auxiliary Data Formats

These formats are generated by analysis pipelines and can be loaded alongside alignments:

| Format | Description |
|--------|-------------|
| Backbone | Conserved genome segments shared across all or a subset of genomes, with multiplicity and positional data. |
| Islands | Non-backbone regions representing genome-specific or subset-specific sequences. |
| Identity Matrix | Pairwise divergence values between genomes. |
| Permutation | Signed permutation representations of genome rearrangements. |
| LCB Coordinates | Left/right boundary positions for each LCB across genomes. |

## Automatic Format Detection

Mauve Viewer detects the file format from the file extension. If the extension is not recognized, the file is treated as FASTA internally, which may result in an error if it is not a valid alignment or annotation format.

## Limits

| Limit | Value |
|-------|-------|
| Maximum file size | 500 MB |
| Maximum entries or blocks | 100,000 |
| Maximum sequence length | 100 MB |

When a file violates these limits, Mauve Viewer displays a descriptive error message explaining which limit was exceeded. No internal error details or stack traces are shown.
